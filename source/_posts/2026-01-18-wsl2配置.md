# github ssh

用之前的方式配置了ssh key，换了个地方，ssh -t git@github ，显示连接关闭了，22端口被干掉了）。换443端口了

[在 HTTPS 端口使用 SSH - GitHub 文档](https://docs.github.com/zh/authentication/troubleshooting-ssh/using-ssh-over-the-https-port)

```
ssh -T -p 443 git@ssh.github.com
```

443端口就可以了。

去.ssh新建一个`config`文件，输入以下内容即可

```txt
Host github.com
    Hostname ssh.github.com
    Port 443
    User git
```

# 导出

使用wsl install和微软商店安装，默认是安装在C盘，可以将其导出为镜像，然后迁移到其他盘再导入，实现迁移

在其他文件夹下输入：wsl --export <发行版名称> <导出的文件名>   // 文件名后缀是tar

卸载：wsl --unregister Ubuntu

导入：wsl --import Ubuntu D:\WSL\Ubuntu(安装位置) D:\WSL\Ubuntu\ubuntu.tar 

设置默认启动发行版：wsl -s 你的发行版名

导入后默认用户为root，需要进行修改

```bash
vim /etc/wsl.conf
然后再里面添加以下信息即可

[user]
default=你的用户名
```

记得关闭wsl再启动，重启才会生效

## 使用

### 复制

尝试将checkhealth的结果复制出来，发现右键不行，显示缺少一个剪切板插件。。。需要在vim里面安装clipboard。

ai给了个简单的方案，就是win的clip.exe剪切板程序,wsl里面也能调用

```bash
将当前缓冲区所有内容输出到剪切板中，但是编码和win有点不兼容，emoji会变成乱码
:%w !clip.exe
```

将结果提供给clip.exe。

Vim内部有很多寄存器，每个都有名字，可以理解为剪切板，用来存各种复杂/删除的内容，我们可以指定从指定的寄存器来读取东西。

不加任何前缀，y、d、p会使用默认寄存器，这些内容不会影响系统的剪切板

常用的寄存器
- `”` 无名寄存器，默认寄存器
- `0` 只存最近一次复制的东西
- `a-z`"命名寄存器等。。
- `+` 系统寄存器
- `*` 选择缓冲区，在wsl里要通过win32yank，clip才能连接到win

### markdown-preview
渲染也需要nodejs什么的，建议装下

在plugin/markdown.lua输入

```
plugin/markdown.lua
return {
  {
    "iamcco/markdown-preview.nvim",
    ft = { "markdown" },
    cmd = { "MarkdownPreview", "MarkdownPreviewStop", "MarkdownPreviewToggle" },
    build = function()
      vim.fn["mkdp#util#install"]()
    end,
    init = function()
      vim.g.mkdp_open_to_the_world = 0
      -- WSL
      vim.g.mkdp_browser = "explorer.exe"
    end,
  },
}


plugin/vim-table-mode
return {
  {
    "dhruvasagar/vim-table-mode",
    ft = { "markdown", "text" },
    cmd = {
      "TableModeToggle",
      "TableModeEnable",
      "TableModeDisable",
      "TableModeRealign",
    },
  },
}
```



然后

```
cd ~/.local/share/nvim/lazy/markdown-preview.nvim/app
npm install
```
