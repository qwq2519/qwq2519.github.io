# 链表与数组


## 链表已死

链表插入O(1),数组O(N),但是这个情况很难成立。因为要删除链表中间节点可能要先遍历很多节点，前提是你持有这个节点的指针，只有不断在首尾操作才是理想状态。

LRU就是一个比较典型

链表的插入删除，还可能涉及内存分配malloc(内存分配器，锁竞争，新节点cache miss等等)，可以通过内存池优化，但是远不如数组方便。

数组，业务常用vector这种，常用reserve可以很大提高性能。

从底层物理硬件特性触发，而不是抽象的数据结构理论。

数组的底层优势很多:
    - cpu cache/访存: cpu有多级缓存，还有个预取器，因此数组在时空局部性上很强,高性能计算肯定都是数组,目前计算机都是冯若依曼架构，目前的计算机架构瓶颈在于访存，cpu计算速度远快于访问内存等，因此数组的memmove什么的计算代价远小于访问内存，多用几次所谓的数组O(n)迁移没想象中的那么慢。  不能忽视内存访问代价 >>> cpu各级缓存 > 计算 ,减少访问内存，避免cpu无事可做才是关键，无事可做就是冯若依曼瓶颈/内存墙
    - 并发: simd等硬件并行，进一步强化数组的速度。而链表一个个节点是不连续的，必须读节点的nxt指针才能指导下一个节点，串行，依赖度高。  遍历vector : 预取器+cpu L1缓存，而 遍历 list可能是 每次指针跳转都是一次 cache miss

归根到底都是要 访存代价>>cpu缓存 > cpu计算

现代编程语言，cpu也在不断优化vector的插入删除，比如c++的右值引用，移动语义，移动代价大大降低，但是list很难优化。

## 链表少数存活空间

crud boy肯定很少用链表了，但是底层系统，一些逻辑性不可避免要用链表，下沉了，不可替代的

### 侵入式链表
（linux代码可以看看，但是看不懂）

普通链表
```
struct Node{
    int data;
    Node* prev;
    Node* next;
}
```
节点包裹数据，数据是节点一部分，要插入数据，不可避免要新建节点，涉及内存分配

侵入式链表
```
struct list_node { struct list_node *prev, *next; };

struct Task {
  int pid;
  struct list_node run_queue_link; // 链表指针“嵌”在 Task 里
};
```
业务对象，数据结构自己带着链表节点指针。插入数据时，只是把内部的指针改改，对象集合可能都是在某一块内存上的，新加入数据没有内存分配发生，仅仅只有几个指针的赋值，减少内存分配。更灵活，性能高，但是需要你自己管理内存什么的，更复杂。

Linux内核很多这种嵌入式链表,进程调度队列，驱动设备队列，文件系统缓存等各种对象，这里的对象生命周期复杂，可能存在于多个链表，数组难以表达这种复杂归属

### 无锁数据结构和并发编程

数组的插入，需要移动后面的元素，这个批量移动不是原子的，数组的局部性强，布局紧密反而成为缺点，要线程安全会加个大锁

链表的节点是局部的，要插入节点，只会影响相邻节点，只需要修改相邻节点的指针，这个很适合CAS

Michael-Scott 队列，无锁栈，无锁哈希表都说基于链表节点的原子操作，链表的指针解耦性强，高并发好。

### 内存分配器

操作系统的内存分配，tcmalloc,jemalloc，管理空闲内存块都是用一个空闲链表串起来的，申请内存就是从链表中查找节点并取下。

空闲链表的指针放哪，直接放在空闲内存块里面就行了，原地利用空间。

### 数组模拟链表

用数组模拟链表(算法竞赛里面其实很常见，很多人一直都没用甚至不会指针。 直接开辟一个很大的数组，用数组存索引，通过这个值在连续的数组中查找下一个元素)

这也类似于索引池，内存池。

线段树的 lc[10007] rc[10007] ,val[10007]
各种树也是，trie所有节点都存在 一个vector上，直接存vector下标索引来实现跳转

图的各种存法也是
ringbuffer代替循环队列
堆排序，堆是完全二叉树，用一个数组模拟堆很方便

物理上数组，逻辑上链表
- 内存连续，缓存友好
- 下标访问，链表O1 插入删除
- 下标是一个整数，通常比64位指针小


### 数据库B+树

B+树是矮胖的，是多叉树，一个节点存一页数据而不是一个数据。

页内是有序数组，页外通过链表连接

适配磁盘IO，内存缓存的快大小。

如果是红黑树这种，一个节点几十字节，浪费IO带宽，树深度更大，要进行更多次的访存读盘才能找到目标元素

B树的一个节点既保存了孩子节点的指针，还保存的实际数据的键值对。这就导致了一个节点的孩子节点数量更小。
B+树就干脆把所有的数据值放到叶子节点，这样树结构就更加扁平。并且数据都存储在同一个层级，可以很方便的用一个链表串起来，实现范围查询。


### 块式链表

类似于b+树，链表的每个节点都是一个小数组

类似于算法竞赛的分块，局部用暴力，LSM的一块内数据无序，块间有序等
