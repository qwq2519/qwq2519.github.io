# 提取lowbit

即一个整数二进制最右边的一个1到最低位的部分,即(去除)保留最右边那个1，其他全为0

补码的性质 -x == ~x+1

x    = ....     1000...000
~x   = [高位取反]0111111
-x   = ????     1000...000
x&-x = 0000     1000...000

```C++
int lowbit(int x){
    return x&-x;
}
```


和最低位 1 有关的经典技巧

- 提取最低位的 1：x & -x
- 删除最低位的 1：x & (x - 1)
- 把最低位的 0 变成 1，且其右边全清 0：x | (x + 1)
- 提取最低位 0：~x & (x + 1)
- 判断是否为2的次幂  x&(x-1) ==0

# 其他技巧

## 乘除法优化
编译器会帮我们优化，不用管）,适当减少除法和mod运算，整数比浮点数快，无符号比有符号快等等


## 高维数组
要减少高维数组寻址次数，可以用指针保存上一次用的地址，然后直接加偏移。
a[i]这种[i]运算符就是寻址，如果用指针自增性能会有很大提高

## 消除条件跳转

减少if进入的概率甚至可以消除if（比如可以用空间换时间的做法）

## 循环展开
适当展开，展开太多反而性能下降，因为寄存器不够用，寄存器溢出。

展开后不同语句的并行度提示了，cpu可以利用乱序执行了，而不是原本的强依赖，还涉及流水线，指令预取，分钟预测，寄存器重命名等复杂方面

## 时空局部性（CPU缓存）

- 就是怎么利用CPU高速缓存机制。 像是java程序也需要预热，你一个测量程序跑10次，第二次会比第一次快得多

- 数据对齐：普通变量应该按照它自身的大小对齐，较大的struct应该按照一个2的方幂的边界对齐。避免访问一个小变量小数据，明明一个缓存行放得下，却要跨越边界。

- C的union类型，不仅可以省一点内存，并且里面多个变量在内存的位置是一样的

- 避免动态内存分配，这个没什么好说了）

- 多用局部变量，局部变量都在程序运行时堆栈，这里是数据高频访问区，这里的数据很多都在CPU的缓存里，很快，比全局变量需要32位地址进行寻址快多了。但是不一定，比如大数组，容易导致堆栈溢出，堆栈指针变动幅度大，反而变成负面优化

- struct of arrays与arrays of struct：看情况吧，看你怎么访问，不一定，矩阵运算和存多维点可能就不一样了。

- 空间：如果你访问了某个内存地址，附近的数据很可能已经被一起读进缓存了。尽量使用步长为1的访问模式，特别是高维数组。


- 时间：如果某段数据刚刚访问过，很可能很快还要再访问。短时间内会反复用到的数据尽量留在缓存中。使得内存访问的工作集尽量小（太大的话缓存放不下），甚至能复用。 比如统计整数二进制表示中1的个数，查表法，分两段有时不如分三段

- 尽量避免步长为较大的2的次幂，避免缓存冲突。（CPU 内存: 缓存这个映射规律,怎么说，导致缓存频繁失效冲突吧）。算法竞赛的数组大小一般都不开2的次幂大小，都是加上个小数，质数大小最好，这叫padding。特别是高维数组的每一维的大小

## 代码缓存
上面是CPU数据缓存，这里讲代码缓存，代码也是放在内存的，访问频率可能比数据更高，原理也是CPU高速缓存差不多

- 相关联的函数放在源代码相近位置

- 减小循环体尺寸：循环耗时不是循环语句，而是内部的循环体语句，可以把一个循环体内部比较独立的语句拆出来，比如拆为多个循环。对于复杂的，大的循环体拆了会有很大的提升,对分支预测也友好


总结起来，对于CPU 缓存类型的优化，我们编写代码的大体原则就是：本来就有关联的东西（变量、代码），尽量在代码组织上让它们紧密地联系起来。本来毫无关联的东西，就应该把它们相分隔开。这不单单是可以让我们的代码更加有效率，同样也是编写更优美的代码的基本原则。 


## 分支预测

高级语言各自if 循环，汇编层面就是个二路跳转，当cpu某个位为set的时候，就跳转

条件跳转是效率低下的，因为现代指令一条能拆成十多个步骤，cpu为了提高效率就采用了流水线，还引入了分支预测，而分支跳转会取消的分支预测，还要回滚，浪费之前预测的大量指令周期。

底层汇编采用了大量消除条件分支的写法，我们高级语言变成也可以做一些类似的，提高分支预测准确率。高级语言的if可能不止对应底层汇编的一条条件语句，因此if里面的条件尽可能精简

### 寄存器重命名

有两条不相关的指令，但是用到了同一个寄存器，前者可能因为访存而阻碍后者？ 但是如果每个指令都单独一个寄存器，寄存器不够用啊

其实现代处理器的寄存器有有很多了，eax ebx什么的是逻辑处理器，只是一个概念。cpu会有一个寄存器映射表，到了uop级别，会将eax什么映射到物理寄存器上

这样就能乱序执行了
