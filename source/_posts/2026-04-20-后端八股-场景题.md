# 场景题

## 长短链系统

### 需求

- 核心目标：将长链接转换为短链接，用户点击短链后，快速稳定的302 301跳转到原始长链
- 基本功能：生成/自定义短链，短链跳转，过期时间，统计点击量，风控防刷，批量生成等
- 非功能目标：高可用，低延迟，高并发，可扩展（支持统计分析，AB投放等扩展能力）

合理假设:互联网场景通用短链平台，读多写少，跳转流量远大于生成流量

### 架构

- 短链生成服务：负责校验长链，生成短链，写库写缓存
- 短链跳转服务：根据短链查库，完成302 301跳转
- 存储层：mysql redis
- 异步链路：MQ做点击日志，统计，风控等

其他的就CDN，边缘节点什么了

### 核心流程

1. 短链生成

全局唯一ID+base62编码

- 冲突概率低
- 编码后长度低
- 生成性能高，不需要多次重试查重
- 扩展方便

为什么不用长链hash截断呢，这样容易冲突，还容易暴露规律，hash方案可以作为去重辅助键。害怕慢就加个布隆过滤器咯。

怕被猜出规律就对id先做一次可逆变换再base62。base62不是哈希算法，是编码方式，不会有哈希冲突。

随机串优点是不易枚举，但是有碰撞，需要重试。
hash长链可以做同长链复用，但是冲突，可以用来辅助去重

2. 短链跳转

- 解析短链的code，查redis，命中直接302
- 没命中就查mysql，然后回填redis，返回302
- 返回404或者统一页面
- 短链过期什么的统一返回状态页

天然读多写少，采取cache aside策略。短链一旦生成很少改了，缓存一致性压力很小，失效和封禁需要立刻生效，因此需要主动删缓存，多节点可能要使用订阅通知机制。

高可用就是redis集群，哨兵等。mysql主从，读写分离，但跳转一般优先查主，避免主从延迟。

高并发：热点短链可能就是需要redis，本地缓存，网关缓存，CDN缓存等。 还要根据访问频率自动发现热点，自动预热什么的。

还要什么缓存穿透，击穿，雪崩，经典八股了。

分库分表，分片按照id水平分片，短链里面可以编码分片信息，或者简历全局路由表

302 better than 301
- 302更灵活，如果长链要变更，灰度，风控，更不容易被浏览器强缓存
- 301适合永久固定连接，但一旦被客户端缓存，后续治理成本高

### 存储

1. 主表：短链映射表
id 短链 长链 长链hash 过期时间，状态 等

2. 统计表
点击日志肯定要异步化，不能阻塞跳转主链路

异步化流程
- 跳转成果投递mq或者写日志
- 消费端做各自统计
- 明细数据可以落es 等分析性存储等

## 高并发下单接口的幂等机制

需求：为什么需要幂等
- 手抖连点
- 前端重试
- 网关超时重试
- 消息重复投递

目标：同一业务请求只能成功创建一次订单；下单链路高并发且不能出现重复订单，重复扣减库存，重复支付等

架构
- 接口层：业务幂等号
- 服务层：状态机校验
- 存储层：唯一索引兜底
- 异步层：MQ消费幂等

核心流程
1. 前端生成或服务端下发幂等号requestID，用户进入确认页或者提交订单时带上这个id
2. 服务端先做幂等校验（使用redis setnx requestID value expire）
3. 执行业务逻辑
4. 数据库唯一索引：订单表可以设计（userid，bid）或requesetID等唯一索引，避免redis失效
5. 返回历史结果（如果是重复请求，可以返回历史

  

## 智力题



### randM()实现randN()
0. M>=N的话，直接不断重试就行了。
1. 找一个最小整数k，使得Mk>=N
2. 调用k次randM()，生成[0,M^k-1]之间的数
3. 找到小于M^k的N的最大倍数 limit=floor(m^k-1)/N)\*N
4. 如果val<limit，返回val%N
5. 否则重试

```c++
int randN() {
    long long range = 1;
    int k = 0;
    while (range < N) {
        range *= M;
        k++;
    }

    while (true) {
        long long x = 0;
        for (int i = 0; i < k; i++) {
            x = x * M + (randM() - 1);  // 0..M-1
        }

        long long limit = (range / N) * N;
        if (x < limit) {
            return x % N + 1;
        }
    }
}
```
这样拼出来才是等概率的，并且保留limit个数，恰好是N的倍数

总结：进制转换+拒绝采样。
